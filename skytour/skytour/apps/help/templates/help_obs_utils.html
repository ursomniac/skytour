Starting a New Observing Session

Each Observing Session ("Run") takes place on a given night at a given location.

  1. Navigate to Session üîΩ and then Start New Session
  2. Fill in the form. Note that you're entering the UT Date (not local time)!
  3. Then the new session will appear under Latest/Active Session in the same drop-dropdown.
  4. If you then add new observations and/or observing conditions around the same time as the Observing Date, the forms SHOULD populate any "Session" field with the correct session (you can override this if you need to).

Creating an Observing List

You can create observing lists to organize which objects you want to prioritize.

  1. From DSO Lists üîΩ go to All DSO Lists
  2. There, click on give it a name and description and it will be added.

If you are running multiple telescopes, having separate lists for each can be very handy.

Active Observing Lists

If the observing list is marked "active", then those objects will be tagged on other DSO lists (with the üî≠ icon).

You can make a DSOList "inactive" (for example if the objects that were on it aren't likely to be available soon, but you want to keep the list around for the future).

Adding DSOs to a DSO List

On almost every table that has a list of DSOs there are buttons for "Add to DSO List". Select the list, and it'll be addded.

Likewise on every DSO Detail page, there's a section showing:

  • Which DSO Lists the DSO is a member of.
  • A link to add it to a DSO List

@Now and @Cookie

These two functions display a filtered list of DSOs that are above the horizon for the current location:

  • @Now uses the current UT date/time
  • @Cookie uses the UT date/time of the cookie

On the page, there's a form where you can customize the list:
UTExplicitly set the UT date/time (doesn't affect the cookie)
UT OffsetSet an offset for the UT date/time1
Min. Priority(default = 3-High) - set the minimum priority (for the observing mode)
Min/Max DecSet the range of allowed declinations2
Min/Max AltSet the range of allowed altitudes (default is what's set in the cookie)
ImagedExclude objects you've already seen3
GearSelect a sample of Observing Modes
No MaskIf a mask has been set for your location, ignore it
On Active DSO ListOnly include objects already on an active DSO List
Show ThumbnailsShow a small thumbnail image in the list4

Notes
  1. Helpful to "nudge" the list, e.g., "What is going to be up in an hour?"
  2. Typically you don't need to adjust this, as you can also set Min/Max Altitude which is usually what you do want to do.
  3. The options are:
    • All - show everything
    • Yes - show ONLY things you've imaged
    • No - show ONLY things you've NEVER imaged
    • Redo - but DO include DSOs with the "Reimage" flag set
  4. The thumbnail is chosen in this order:
    1. A library image you've taken
    2. An uploaded external image
    3. The Skytour logo
    Note that turning thumbnails on slows down rendering of the page slightly.

Making Observations

You can add a new observation to your session in three ways (they all go to the same form):

  1. Click Add Observation: SSO/DSO in the Navbar.
  2. On any object's (DSO/Planet/Comet/Asteroid) Detail Page, there will be an button.
  3. Likewise on the Detail page for any Session, there is also an button.

The Observation Form

Although it's a little complicated, for the most part there is very little you have to enter:
UT Date/UT Time The date/time of the observation - these will be filled in with the current UT date/time.
Object Type This is one of:
  • DSO
  • Planet
  • Comet
  • Asteroid
  • Moon
  • Other
For Planet/Asteroid/Comet - choose from each of their respective dropdowns. (For Moon you don't have to select anything.)
For DSOs, you then need to select:
DSO Catalog / ID in Catalog Select the appropriate Catalog (e.g, "M" for Messier), and then enter in the ID in that catalog (e.g, "44" for Messier 44).
The back-end IS "smart enough" to handle aliases (so either "C 6" or "NGC 6543" will both work); DSO in Field IDs "should" work (so, NGC 5195 "should" store the observation under "M 51", though I haven't rigorously tested this).
Session(see above)
Telescope(defaults to the "default" telescope)
Location(defaults to the cookie location). This is here mostly for the situation where you "forgot" to add an observation, and only remember the next day or something.
(Typically you only observe from one location on a given night, but early on, I had a weird situation where I was testing conditions at several sites... so...)
Eyepiece/FilterFor optical observing, though you can use this for a "smart" scope if they have a dual passband filter.
Notes(self explanatory)
NOTE: if you get to this form from an object's Detail page the identifier fields (DSO/Planet/etc. and their dropdowns or "ID in Catalog") will already be filled out for you!

Recording Observing Conditions

If you have the equipment:

  • A thermometer for the temperature
  • A hygrometer for relative humidity
  • An anemometer for wind speed
  • A light meter for measuring sky brightness
or if you're just relying on apps for the local weather conditions, you can record these measurements as part of your session.

Navigate to Conditions to do this.

SQM Measurements

Skytour will generate statistics based on your measurements of sky brightness. However, you might want to filter some of them out from averages and distributions:

  • Use SQM in Stats if set to "No" will record the measurement, but not use it in averages/stats.
  • Checking Moon flags measurements taken when the Moon is up (it won't flag this automatically).

Skytour will estimate a fractional Bortle value (sort of) from this.
NOTE: you're likely to find that the actual measures are about "1/2 Bortle" worse than what you'll find on apps for your location. This is likely do to encroaching light pollution since the time the maps were last generated. ‚òπÔ∏è
